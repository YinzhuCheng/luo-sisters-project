Overview

总览

企划的前台是少女生活与姐妹互动；中层是秩序与无序的相互拉扯；深层是当工具越来越会给答案，人如何保留自己的选择、关系与经历。

洛青悠

用秩序保护梦想的人。

青绿色古风 Lolita、手账、相机、茶点、路线图。她会把别人随口说出的愿望认真写下，再温柔地推向现实。

洛有栖

用无序寻找梦想的人。

蓝白 JK Lolita、钥匙、白兔、怀表、秘密门。她把普通校园看成童话入口，总能发现别人错过的奇妙线索。

企划主线

兔子洞茶会。

姐妹筹备一场带 Alice 气质的小型茶会。茶会从可爱的日常目标，逐渐变成两人面对焦虑、选择与时代工具的试炼。

Layer 01

人设层

人设按八层拆解：第一眼、日常、行为记忆、能力、关系、焦虑应对、隐藏矛盾、成长。这样角色先作为“人”成立，主题随后从她们的选择里长出来。

洛青悠
规划、记录、现实、古风 Lolita
一句话定位用秩序保护梦想的人。
前期记忆句“那就给这场梦订个日期吧。”
照顾型口癖“茶和退路，我来准备。”
成熟后信念计划为了让迷路之后还能找到彼此。
1. 第一眼层

青绿色古风 Lolita、青衿感、云肩、盘扣、蕾丝、旧相机、手账、茶点。她给人的第一印象是温柔可靠，好像只要她在，事情就能慢慢收拢。

2. 日常层

她会提前抵达，包里常备发夹、创可贴、胶带、便签与小剪刀。她拍照前先看光，再看人；喜欢旧街、茶馆、布料店、文具店、旧相机店。

她把票根、干花、糖纸、拍立得夹进手账。她对“随便”会短暂停顿，因为她想知道该如何准备。

3. 行为记忆层

听到有栖的新点子，会先微微睁大眼，再翻开青绿色手账。紧张时整理袖口或相机带；看到好看的光会突然停下；写计划时会留一个小栏，名字叫“意外”。

4. 能力层

她擅长规划、统筹、审美判断、资料整理、拍摄记录、活动执行、风险预案。她能把“迷路的人才能参加的茶会”拆成场地、路线、邀请函、茶点、服装、拍摄与雨天方案。

5. 关系层

有栖的幻想，是青悠生活里最亮的意外。青悠会保护、准备、陪伴，也会在紧张时把保护推进成安排。她需要学习给有栖的梦保留呼吸空间。

6. 焦虑应对层

青悠用秩序对抗焦虑。她相信整理、记录、路线与备用方案能让生活少一些散落感。她的温柔里带着疲惫，因为她常把“照顾大家”写进自己的责任清单。

7. 隐藏矛盾层

她早期反感 AIGC，表层理由是生成物太快、太整齐、缺少经历。更深的情绪来自价值感动摇：如果工具也能整理、规划、优化，她擅长的事还剩下什么重量？

8. 成长层

她从“亲手完成全部步骤”走向“选择该亲手保留的部分”。她会使用 LLM 与 AIGC 搬开重复劳动，把精力留给判断、关系、审美与共同经历。

最终状态：有弹性的秩序。她仍然规划，也会给未知留下位置。

洛有栖
幻想、入口、Alice、JK Lolita
一句话定位用无序寻找梦想的人。
前期记忆句“姐姐，你有没有觉得，这扇门今天不太一样？”
核心道具句“钥匙还在，所以门一定也在。”
成熟后信念这一次，她想亲手打开门。
1. 第一眼层

蓝白 JK Lolita、白衬衫、背带裙、领结、雏菊、钥匙项链、白兔挂件、怀表包、旧书。她像在校园里寻找兔子洞的 Alice 气质少女。

2. 日常层

她会在走廊尽头停下来，看一扇普通的门。书包挂件很多，走路时轻轻作响。她喜欢旧图书室、天台门口、文具店、甜品店，日记常写成故事开头。

3. 行为记忆层

思考时捏钥匙项链，紧张时摸领结；看到门、镜子、楼梯、窗户会多看一眼。她会把普通物件说成童话线索，把青悠手账边角画满兔子、门与箭头。

4. 能力层

她擅长概念创造、氛围命名、童话式表达、发现日常里的异常感。她写出的邀请函句子会带着奇妙钩子：“请携带一枚迷路，在茶凉之前抵达。”

5. 关系层

她依赖青悠的执行力，也渴望自己的奇怪想法被保留下来。她希望姐姐相信她看见的门有意义，同时也希望自己拥有选择门的勇气。

6. 焦虑应对层

有栖用探索对抗焦虑。面对压力，她会绕路、发呆、看门、写奇怪的句子，让答案晚一点抵达。这样的自由保护了她，也让她容易停在“如果有一天”。

7. 隐藏矛盾层

她害怕现实配不上想象。想象中的茶会永远美丽，现实里的茶会会下雨、迟到、烤焦、预算不足。AIGC 生成出的完美梦境会在中期吸引她。

8. 成长层

她从等待白兔带路，走向自己选择门。她保留幻想，也开始承担结果；保留迷路感，也愿意写下第一张邀请函。

最终状态：有锚点的自由。她仍会迷路，也会亲手出发。

姐妹关系公式

有栖给青悠一个梦，青悠给有栖一条路。

青悠给有栖现实、路径、抵达的可能；有栖给青悠幻想、惊喜、出发的理由。两人的成长要靠彼此推动。

关系推进

前期：轻喜剧式互补。
中期：可爱的习惯变成压力。
后期：一个学会留空白，一个学会做选择。

互动样例

有栖：姐姐，我想办一场只有迷路的人才能参加的茶会。
青悠：那就给这场梦订个日期吧。
有栖：迷路的人怎么知道日期？
青悠：所以邀请函要提前寄出。还有一条看起来会迷路、实际能回来的路。
Layer 02

故事梗概层

故事表层是一场兔子洞茶会的筹备；中层是姐妹之间的秩序与无序；深层在中后期露出 AIGC 对焦虑、陪伴、选择与价值感的放大作用。

表层主线

姐妹要举办一场带 Alice 气质的小型主题茶会。准备过程包含找场地、设计服装、写邀请函、拍宣传照、做点心、布置入口与路线。

情感主线

青悠希望茶会安全落地，有栖希望茶会保留神秘。一个想把梦稳稳托住，一个想让门后继续发光。

时代暗线

AIGC 作为生活背景逐渐显影：工具、陪伴、顾问、推荐系统。它帮助角色，也放大每个人面对焦虑时的习惯。

章节推进

阶段台前事件人物变化AIGC 露出方式
开篇有栖提出“迷路茶会”，青悠开始记录。姐妹互补关系建立，读者先喜欢她们的日常。同学随口用 AI 生成海报，像日常工具。
筹备前期找旧图书室、做邀请函、设计蓝白与青绿两套服装。青悠的可靠与有栖的灵气同时发光。白兔系统给出参考图与路线建议。
小刺痛青悠手作草稿被 AI 版本轻松压过。青悠轻微沉默，价值焦虑埋线。AIGC 成为效率对照。
分歧青悠用 LLM 优化邀请函，童话句子被改得很标准。有栖感到自己的“门”被关上；青悠感到自己被误解。工具成为误会的导火线。
中段危机茶会越来越完美，也越来越空。有栖短暂沉入生成 Wonderland。青悠困在任务清单，有栖困在完美梦境。AIGC 作为生成、陪伴、顾问全面浮现。
转折青悠承认自己害怕失去价值；有栖承认自己害怕打开门。两人开始共同写提示词与规则。AI 回到辅助位置，边界由角色决定。
结尾雨天茶会开场，流程有意外，记忆因此成立。青悠留空白，有栖做选择。工具退到后台，真实经历站回台前。

AIGC 登场节奏

前期｜5%

作为生活空气：海报、参考图、时间表，轻轻出现。

中前期｜15%

成为效率对比：青悠感到轻微刺痛。

中期｜35%

配角展示陪伴、顾问、生成三种用法。

中后期｜45%

直接放大姐妹冲突，推动观念转换。

后期｜20%

退回工具位置，人物关系与真实经历收束主题。

配角与 AI 使用风格

南星｜AI 作为陪伴

她有一个长期聊天对象“茶匙先生”。它稳定、记得她说过的话、给她安慰。她的成长是从稳定回应走向真实关系的轻微冒险。

明夏｜AI 作为顾问

学生会型角色，习惯让系统评估风险、流量与路线。她代表“最优解”的诱惑，帮助青悠意识到最稳的路未必最想走。

白兔系统｜时代入口

推荐、生成、提醒、引导的集合体。它很有用，也很会催促。它的口吻可以轻快：“你将错过下一扇更适合你的门。”

关键场景钩子

  • 有栖写下邀请函：“请携带一枚迷路，在茶凉之前抵达。”
  • 青悠用 LLM 润色后，句子变得正式清楚；有栖说：“它把我的门关上了。”
  • 有栖对青悠说：“姐姐，你把迷路也安排好了。”
  • 青悠第一次承认：“我怕这些都能被替代以后，我也变得可有可无。”
  • 姐妹共同改写提示词：保留迷路感，保证客人可以安全回来。
  • 雨天茶会当天，原计划之外的水洼、伞、错路与笑声成为真正的记忆。
Layer 03

外观层：视觉提示词与制作方向

这一层先写提示词与制作规格，后续再逐步生成图像。提示词用于提高项目可控性：先控角色识别点，再控线稿、立绘、三视图、表情、服装、配饰、物件、姿势与代表 CG。

统一美术方向

风格：日本二次元角色设定图，清洁线稿，柔和赛璐璐上色，细节清楚，色彩低饱和，带轻小说封面与角色设定资料感。

画面控制：服装结构清晰、配饰可复用、发型剪影稳定、颜色用途明确；每张设定图优先服务后续复刻与统一。

文字制作规范

生成阶段：优先生成角色、服装、框图、装饰与留白，正文与标签尽量保持少量或留出空位。

排版阶段：标题、信息栏、表情标签、物件说明与长段背景文字，使用 HTML / SVG / PIL 等工具后期叠加，保证中文清晰、字号统一、框内不溢出。

成品策略：先控画面，再控文字。角色图像由绘图模型完成，文字信息由排版工具完成。

表情差分硬性规范

每个表情必须同时改变眉形、眼神、嘴型、头部角度、手势或肩部姿态，避免只改标签与轻微嘴角。

青悠的表情建议加入：温柔、闭眼微笑、专注记录、侧目思索、轻微惊讶、困扰微笑、疲惫温柔、雨后释然。

Production Workflow

日本二次元角色企划制作规范

后续制作建议按“概念 → 线稿 → 立绘 → 设定图 → CG → 应用素材”的顺序推进。每一步都保留可检查标准，便于修正与统一。

阶段产出检查重点
0. 角色圣经姓名、核心句、主题功能、关系、禁改识别点。一句话能讲清她是谁，三处记忆点稳定。
1. Moodboard服装、发型、配色、材质、道具、场景参考。青悠偏青绿古风与茶；有栖偏蓝白校园与 Alice。
2. 剪影草案每人 3–6 个小剪影。遮住细节后仍能区分两人。
3. 线稿探索发型、裙摆、领口、袖口、鞋袜、包袋结构。服装层次清楚，配饰位置稳定。
4. 彩色小稿3 组配色与明暗测试。主色、辅助色、点缀色用途明确。
5. 标准立绘全身立绘，正面站姿，透明或浅色背景。可作为后续所有图的母版。
6. 三视图正面、侧面、背面，含关键结构说明。裙摆、发饰、包带、鞋袜、背面蝴蝶结完整。
7. 表情差分每人 8–12 个表情，要求眉、眼、口、头部角度、手势同时拉开。基础性格、隐藏焦虑与成长后的松弛感都能表现。
8. 姿势设定站姿、坐姿、拍照、写手账、看门、递邀请函。动作服务角色，不只追求漂亮。
9. 服装变体标准服、日常服、茶会服、雨天服、冬季服、轻便校园服。换装后保留核心识别点。
10. 配饰物件相机、手账、钥匙、怀表包、邀请函、茶杯、书包。道具能独立讲故事。
11. 代表 CG每人 2 张个人 CG，加 1–2 张双人 KV。CG 要表现角色主题、关系与成长瞬间。
12. 应用素材头像、Q版、表情包、社媒横幅、Live2D 拆件草案。用于宣传与后续迭代。

下一步建议

为了让项目可控，建议先从标准化资料开始，再进入图像生成与人工修正。

1. 固定角色卡

确认年龄、身高、生日、学校/社团、家庭关系、说话方式、常用道具。

2. 做第一版三视图

优先生成线稿或低色彩草图，先看剪影与结构，再逐步增加材质细节。

3. 确定第一张双人 CG

建议从“门与手账”开始：有栖发现门，青悠记录并准备退路。
